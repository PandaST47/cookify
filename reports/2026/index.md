# Отчёты Cookify за 2026

Хронологический индекс технических changelog'ов по проекту Cookify за 2026 год.

## Май

- [2 мая — Bootstrap CLAUDE.md + 36 рецептов + полная фильтрация + таб «Приготовлено»](./week3_05-01_to_05-07/05-02_changelog.md)
  Большой стартовый день: создан проектный гайд `CLAUDE.md` (13 разделов
  + maintenance log) по эталону DirectStudio; справочник рецептов расширен
  с 6 → **36** записей с полным покрытием всех фильтров; добавлены три
  новых хука (`useCooked`, `useRatings`, `useFilteredRecipes`)
  с per-user persistence и аггрегатной оценкой. Реализована полная
  иерархия фильтрации из ТЗ (sort → ingredients → exclusions → sidebar
  group AND/OR + cross-group AND), включая правило «явно выбранный
  ингредиент бьёт исключение из профиля». Воссоздан макет «Приготовлено»
  один-в-один со скриншота: 3-колоночные строки с интерактивной шкалой
  ⭐⭐⭐⭐⭐, кнопками «Отправить оценку / Изменить оценку / Удалить»
  и общим рейтингом. Добавлена модалка «Кнопка Ингредиенты» (поиск
  по запятым → ИЛИ-фильтр). `tsc --noEmit` exit 0, `vite build` exit 0,
  bundle 284 kB / 85 kB gzip.

- [19 мая — Сверка с Figma + backend-контракт + code quality](./week5_05-15_to_05-21/05-19_changelog.md)
  Доводка реализованных экранов до 1:1 с Figma-макетом (по детальным
  скриншотам — MCP-доступ к файлу недоступен без Pro-подписки заказчика):
  рейтинг разведён на 3 состояния («Оценить рецепт?» → транзиентное
  «Спасибо!» → «Ваша оценка»/«Изменить оценку»), модалка «Добавить
  ингредиенты» перевёрстана под Frame 69 (кремовая карточка, pill-инпут,
  «Сохранить» справа-снизу), сортировка получила заголовок «Показать
  сначала» + radio-маркеры и точные подписи, в «Праздники» вернулась
  «Показать ещё» (+ Рождество, Другой праздник). Заложен слой
  backend-контракта `services/api.ts` (6 доменов, JSDoc `BACKEND:` с
  REST-эндпоинтами для бэкенд-разработчика). Устранены 2 ошибки линтера
  (sync setState в effect) паттерном «adjust state during render».
  `tsc` exit 0, `vite build` exit 0, **lint 0 errors**, bundle
  285.8 kB / 85.5 kB gzip. Проверено Playwright (7 сценариев).
  **Позже в тот же день:** убран «Поиск по сайту» из Header (#3.1),
  дропдаун сортировки приведён к Figma 426×329 (#3.2). И крупный
  блок #4 — реализованы **3 больших экрана ТЗ 1:1 с Figma** (доступ
  получен через Figma REST API + Personal Access Token): **Полная
  карточка** `/recipe/:id`, **Режим готовки** `/recipe/:id/cook`
  (StepTimer 4 состояния + Советы), **Загрузить рецепт + Черновики**
  `/recipe/new` (форма + автопроверка + уникальность + черновики).
  + lazy-роуты (code-split), кликабельные карточки, `useDrafts`,
  `data/recipeSteps.ts`, `styles/recipe.css`. `tsc`/`build` exit 0,
  lint 0 errors. Проверено Playwright против Figma-рендеров.
