# Cookify — Отчёт по работе · 19 мая 2026

**Проект:** Веб-платформа для кулинарных рецептов
**Стек:** React 19 · TypeScript (strict) · Vite 6 · React Router 7
**Статус сборки:** TypeScript `0 ошибок` · Production build `OK` · ESLint `0 ошибок`

---

## Кратко

За день реализованы **три крупных экрана из ТЗ** строго по макету Figma
(доступ к макету получен через Figma API), плюс полный QA-проход по
безопасности, доступности и адаптивности. Маршрутизация связала ленту
рецептов с детальными страницами.

---

## Что реализовано (1:1 с Figma)

### 1. Полная карточка рецепта · `/recipe/:id`
Открывается кликом по любой карточке в ленте.
- Большое фото-обложка, название, средний рейтинг (с правильным
  склонением «оценка/оценки/оценок»), статус-бейдж «Уже готовили / Новое».
- Кнопка «в избранное», подробное описание.
- Блоки «Пищевая ценность на 100 г» (КБЖУ) и «Время готовки».
- Параметры: время приёма, тип питания, вкус блюда.
- Список ингредиентов со **счётчиком порций** — количества
  автоматически пересчитываются (× порции).
- Кнопка «Перейти в режим готовки».

### 2. Режим готовки · `/recipe/:id/cook`
- Красная кнопка «Выход» → назад к карточке рецепта.
- Пошаговая инструкция: активный шаг подсвечен, будущие — приглушены.
- **Интерактивный таймер на шаге** (4 состояния, как в макете):
  «Начать» → отсчёт с круговым прогрессом → «Пауза/Сбросить» →
  по завершении «Готово / +1 мин».
- Раскрывающиеся блоки «Советы» и «Предупреждение» на каждом шаге.
- На последнем шаге кнопка «Готово!» добавляет блюдо в раздел
  «Приготовлено» и открывает его для оценки.

### 3. Загрузить рецепт + Черновики · `/recipe/new`
- Две вкладки: «Новый рецепт» и «Черновики».
- Форма: загрузка обложки, название, описание, параметры
  (время приёма / тип питания / вкус / праздник / кухня),
  время приготовления, КБЖУ.
- Блок «Ингредиенты» (минимум 3, добавление/удаление, единицы).
- Блок «Пошаговые инструкции» (минимум 3 шага: текст, таймер,
  пояснение, предупреждение).
- Блок «Подытожим»: **автопроверка обязательных полей**, проверка
  **уникальности рецепта** (название + ингредиенты + теги), чекбокс
  «Я проверил(а)». Кнопка «Загрузить» неактивна, пока не выполнены
  все условия — точно по ТЗ.
- «Сохранить черновик» / выход с подтверждением (без потери данных).
- Вкладка «Черновики»: списки «Неопубликованные» и «Опубликовано»
  с датами, открытием и удалением.

## Качество (проверено, не на словах)

| Критерий | Результат |
|---|---|
| **Безопасность** | Нет `dangerouslySetInnerHTML`; весь текст экранируется React. Загрузка обложки: только растровые форматы (**SVG отклоняется** как потенциально скриптовый), лимит 4 МБ, проверка содержимого. Черновики из хранилища валидируются по структуре. |
| **Чистый код** | TypeScript strict, **0 `any` / `@ts-ignore`**, ESLint 0 ошибок. Переиспользованы существующие хуки и паттерны проекта. |
| **Оптимизация** | 3 новых экрана подгружаются лениво (code-splitting) — не утяжеляют стартовый бандл. `memo`/`useMemo`/`useCallback` в нужных местах, корректная очистка таймеров. |
| **Доступность** | Семантические теги, `aria-*` на обязательных полях, озвучка ошибок (`role="alert"`, `aria-live`), модалки с Esc и фокусом, навигация с клавиатуры. |
| **Адаптив** | Проверено Playwright: **0px горизонтального скролла** на 375px и 320px на всех страницах. Раскладки корректно перестраиваются. |
| **Соответствие Figma** | Доступ к макету получен через Figma API; верстка сверена с рендерами. |

## Честно: что упрощено

Чтобы не вводить в заблуждение — редактор отдельного шага в форме
загрузки в Figma богаче (фото на каждый шаг, поле «использованные
ингредиенты», «кулинарные термины», «Оборудование»). Реализована
**функциональная суть по ТЗ** (текст + таймер + пояснение +
предупреждение). Расширение до полного редактора шага и точные фото
к шагам — запланированы (backlog #8.5–#8.6) и не влияют на работу
ключевых сценариев.
