- 🟡 Should — делаем после must
- 🟢 Could — делаем когда будет время

- 🎨 visual / 🧠 logic / ♿ a11y / 🚀 perf / 🔒 security

## 🏆 Milestone 1 — Профиль (завершить по Figma)

Профиль сейчас работает функционально, но по дизайн-моментам Figma есть что добить.

### 1.1 🔴 🎨 Сайдбар разделов в профиле ⏱ 2h

В Figma у профиля обычно есть левая колонка с пунктами:
```
┌─────────────────────────────┐
│ Мой профиль         ▍       │
│ Мои рецепты                 │
│ Избранное                   │
│ Приготовлено                │

│ Настройки                   │
```

**Реализация:**
- Новый компонент `components/profile/ProfileSidebar.tsx` с nav-элементами

- На мобилке сворачивается в горизонтальный скролл-табс
- Ссылки роутятся — каждый пункт отдельным subrouте: `/profile`, `/profile/recipes`, `/profile/favorites`, etc.

**TypeScript:**
```tsx

<Routes>
  <Route path="/" element={<ProfileLayout />}>
    <Route index element={<ProfileMain />} />
    <Route path="recipes" element={<ProfileRecipes />} />
```

### 1.2 🔴 🧠 Страница «Мои рецепты» ⏱ 4h

CRUD рецептов пользователя.

- Карточки рецептов (переиспользуем `RecipeCard` из Cookify)

- Empty state: «У вас пока нет рецептов» + иллюстрация + CTA «Создать первый»

**Хранение:** `localStorage` ключ `cookify:userRecipes:<userId>`. Новый сервис `recipeService.ts` по образу `mockAuth.ts`.

- Сердечко на каждой `RecipeCard` (уже есть место под него в дизайне)

### 1.4 🟡 🎨 Изменение аватара с кроп-окном ⏱ 3h

Сейчас: загружаем фото как есть (по клику на «Загрузить фото»).
Надо: после выбора файла открывать модалку с кроппером, чтобы юзер мог выбрать квадратную область.

### 1.5 🟢 🎨 Прогресс регистрации на странице профиля ⏱ 1h

«Ваш профиль заполнен на 60%» с подсказками:

### 1.6 🟡 🧠 Настройки профиля ⏱ 3h

Страница `/profile/settings`:

## 🏆 Milestone 2 — Функционал рецептов (Figma + аналитика)

### 2.1 🔴 🧠 Детальная страница рецепта ⏱ 6h

`/recipe/:slug` — полная страница с рецептом.

**Структура (по типичной Figma для кулинарных приложений):**
1. Hero: большое фото + название + рейтинг + время + калории
2. Ингредиенты (слева) — чеклист, можно отметить что уже есть, пересчёт пропорций
3. Шаги приготовления (справа) — пронумерованные, с таймером для каждого («10 мин тушим»)
4. Автор рецепта + дата
5. Отзывы/комментарии
6. «Похожие рецепты» внизу
7. Кнопки: «Добавить в избранное», «Приготовлено», «Поделиться»

**URL как slug, не id:** `/recipe/syrnyy-krem-sup` — SEO-friendly. Нужна утилита `slugify(title)`.

**React Router 7:** loader → префетч данных рецепта перед рендером. Для мока — синхронно из `recipes.ts`.

### 2.2 🔴 🎨 Загрузка рецепта (multi-step wizard) ⏱ 8h

`/upload` или модалка. Пошагово:

**Шаг 1 — Основное:**
- Название
- Описание
- Обложка (upload + crop)
- Категории (multi-select: ПП, завтрак, обед, ужин...)
- Время приготовления
- Порции

**Шаг 2 — Ингредиенты:**
- Динамический список (add/remove rows)
- Название + количество + единица измерения
- Автокомплит из базы ингредиентов (later: tag-based поиск)

**Шаг 3 — Пошаговое приготовление:**
- Repeatable blocks: текст шага + опциональное фото + опциональный таймер в минутах
- Drag-and-drop для переупорядочивания (`@dnd-kit/core`)

**Шаг 4 — Питательная ценность:**
- Авто-расчёт на основе ингредиентов (для моки — заглушка: «автоматически рассчитается»)
- Можно руками задать КБЖУ

**Шаг 5 — Предпросмотр и публикация:**
- Как будет выглядеть карточка в списке
- Как будет выглядеть страница рецепта
- «Сохранить как черновик» / «Опубликовать»

**Стейт:** `useReducer` для формы + Zustand/Jotai если появится потребность в глобальном флоу. Пока `useReducer` достаточно.

### 2.3 🟡 🧠 Отметить как приготовленное + трекинг ⏱ 3h

- Кнопка «Приготовлено» на карточке/странице рецепта
- В профиле — `/profile/cooked` со списком приготовленного + дата
- Для каждого — возможность оставить свой отзыв (рейтинг + комментарий «что получилось»)
- Статистика: «Вы приготовили 12 блюд в этом месяце» (в шапке профиля)

### 2.4 🟡 🧠 «Мои продукты» (ингредиенты в холодильнике) ⏱ 5h

- Страница `/profile/ingredients` (или переименуем «Мои продукты»)
- Список ингредиентов которые есть у юзера (добавить / удалить / отметить как закончилось)
- **Главная фишка:** на основе «моих продуктов» — автосортировка рецептов на главной по максимальному совпадению
- Бейдж «У вас есть все ингредиенты» на карточках рецептов
- Бейдж «Нужно докупить: X, Y»

Это совпадает с полем `hasAllIngredients` в типах — оно уже есть, просто не используется.

## 🏆 Milestone 4 — Auth completion

### 4.1 🔴 🔒 Восстановление пароля ⏱ 3h

Сейчас кнопка «Забыли пароль?» → `alert()` заглушка. Надо:

- `/forgot-password` — форма с email
- Отправить код (как в register step 2)
- `/reset-password` — ввод нового пароля (2 раза) + кода
- В `mockAuth.ts` добавить `requestPasswordReset(email)` и `resetPassword(email, code, newPassword)`

### 4.2 🟡 🔒 Подтверждение email после смены ⏱ 2h

Если юзер меняет email в профиле — должен подтвердить новый через код (чтобы нельзя было увести аккаунт в соседнюю почту). Сейчас меняется сразу.

### 4.3 🟢 🔒 OAuth (Google / VK / Yandex) ⏱ 4h (требует бэкенд)

- Кнопки «Войти через Google» / «через Яндекс» на login/register
- Для моки — эмулируем OAuth-дансе: клик → искусственная задержка → auto-создание юзера с префиксом `google_user_xxx@`
- Для прода — делается на бэкенде, тут только UI

### 4.4 ⚪ 🔒 Двухфакторная аутентификация ⏱ 6h (v2)

TOTP через Google Authenticator / Authy. Полностью бэкенд-завязанная фича, без сервера не имеет смысла даже моку писать.

### 4.5 🟡 🔒 Rate-limiting visible feedback ⏱ 1h

Сейчас в mockAuth нет rate-limit — можно бесконечно ресендить коды (если не countdown на клиенте).

- В `mockAuth.ts` — счётчик попыток за последние 10 минут
- После 5 резендов — блок на 15 минут
- В UI — понятное сообщение «Слишком много попыток, попробуйте через 14 минут»

## 🏆 Milestone 5 — UI / UX / a11y полировка

### 5.1 🔴 ♿ Полный клавиатурный тест ⏱ 2h

Пройти все сценарии **только с клавиатуры** (Tab / Shift+Tab / Enter / Escape):
- Логин ✓ (протестировано)
- Регистрация ✓
- Профиль: edit-on-click ✓, модалка ✓
- Главная: фильтры, сортировка, переходы на рецепты — **не тестировал**
- Sort dropdown (в HomePage) — **известная проблема**, работает только мышкой, нужен ArrowDown/ArrowUp/Enter

### 5.2 🔴 ♿ Focus trap в модалках ⏱ 2h

Сейчас в модалке смены пароля `Tab` может «вылететь» из модалки наружу — это баг a11y. Нужен focus trap:
- При открытии — фокус на первый интерактивный элемент
- `Tab` на последнем элементе → возвращается к первому (loop)
- `Shift+Tab` на первом → идёт к последнему

**Либа:** `focus-trap-react` (~4KB) или руками через `useRef` + `FocusEvent`.

### 5.3 🟡 🎨 Скелетоны при загрузке ⏱ 2h

Сейчас: при `await mockAuth.login()` на 600мс — кнопка disabled + текст «Входим…». Это ок для форм.

Для списков/карточек (страницы рецептов, профиль) — **скелетоны** вместо спиннеров (модный подход).

- `components/ui/Skeleton.tsx` — базовый компонент с shimmer-анимацией
- `RecipeCardSkeleton`, `ProfileSkeleton` — использующие его

### 5.4 🟡 🎨 Toasts / Notifications ⏱ 3h

Нужны для:
- «Профиль обновлён»
- «Рецепт сохранён в избранное»
- «Что-то пошло не так, попробуйте позже»
- «Скопировано в буфер»

**Либа:** `sonner` (~10KB) или `react-hot-toast`. Пока — не стал ставить, обхожусь inline-сообщениями.

### 5.5 🟢 🎨 Motion polish ⏱ 3h

`framer-motion` уже в зависимостях — используем только местами. Добавить:
- Stagger-анимацию появления карточек рецептов на главной (уже начали, но не везде)
- AnimatePresence на модалках (плавный exit, сейчас просто убирается)
- Layout animation при drag-and-drop в «Загрузить рецепт»
- Stats count-up на странице «Приготовленного» (цифры набегают)

### 5.6 🟢 🎨 Tema dark mode ⏱ 5h

Вся палитра сейчас на hex в CSS. Надо перевести на CSS-переменные:
```css
:root {
    --ck-bg: #FFFFF7;
    --ck-card: #F9F5E8;
    --ck-accent: #E47809;
    --ck-text: #2C1605;
    /* ... */
}

[data-theme="dark"] {
    --ck-bg: #1C120A;
    --ck-card: #2A1D12;
    --ck-accent: #FF9922;
    --ck-text: #F5EADD;
    /* ... */
}
```

Затем в профиле → настройки → toggle темы.

### 5.7 🟢 ♿ Screen-reader friendly announcements ⏱ 2h

Для успешных действий («Пароль изменён», «Сохранено») — `role="status"` с `aria-live="polite"` — чтобы слепые пользователи слышали подтверждение.

## 🏆 Milestone 6 — Архитектура / Технический долг

### 6.1 🔴 🚀 Декомпозиция Cookify.tsx (монолит 600+ строк) ⏱ 5h

Сейчас все компоненты главной (IngredientsDropdown, RecipeCard, FilterSidebar, TabNav, SearchBar, SortDropdown, ScrollTopButton) живут внутри одного файла. Разнести на отдельные:

```
components/
└── recipes/
    ├── RecipeCard.tsx
    ├── RecipeGrid.tsx
    ├── RecipeCardSkeleton.tsx
    ├── IngredientsDropdown.tsx
    └── FilterSidebar.tsx
components/ui/
    ├── TabNav.tsx
    ├── SearchBar.tsx
    ├── SortDropdown.tsx
    └── ScrollTopButton.tsx
```

Делать **только отдельной итерацией**, не смешивать с другими правками.

### 6.2 🟡 🚀 Lazy-loading страниц ⏱ 1h

```tsx
const LoginPage = lazy(() => import('@/pages/LoginPage'))
const RegisterPage = lazy(() => import('@/pages/RegisterPage'))
const ProfilePage = lazy(() => import('@/pages/ProfilePage'))

<Suspense fallback={<PageSkeleton />}>
    <Routes>...</Routes>
</Suspense>
```

Это уменьшит начальный бандл. Не критично пока проект маленький, но хороший habit.

### 6.3 🟡 🧠 Переход на Zustand для глобального стейта ⏱ 3h

Когда появятся: избранное, мои рецепты, мои продукты, настройки — `AuthContext` начнёт раздуваться или придётся делать ещё 5 контекстов. Лучше сразу Zustand (~1KB gzipped):

```tsx
const useFavorites = create((set) => ({
    items: new Set(),
    toggle: (id) => set((s) => {
        const next = new Set(s.items)
        next.has(id) ? next.delete(id) : next.add(id)
        return { items: next }
    })
}))
```

### 6.4 🟡 🔒 Константы ключей localStorage в одном месте ⏱ 0.5h

Сейчас в mockAuth разбросано: `'cookify:users'`, `'cookify:session'`, `'cookify:pending'`. Выносим в отдельный `services/storageKeys.ts`:

```ts
export const STORAGE_KEYS = {
    USERS: 'cookify:users',
    SESSION: 'cookify:session',
    PENDING: 'cookify:pending',
    FAVORITES: (userId: string) => `cookify:favorites:${userId}`,
    USER_RECIPES: (userId: string) => `cookify:userRecipes:${userId}`,
    MY_INGREDIENTS: (userId: string) => `cookify:ingredients:${userId}`,
    SETTINGS: 'cookify:settings',
    THEME: 'cookify:theme',
} as const
```

### 6.5 🟢 🚀 Вынести `contexts/` на уровень `src/` ⏱ 0.5h

Семантически контексты — не UI-компоненты. Но сейчас важнее функционал, чем микро-вкусовщина. Отложим.

### 6.6 🟡 🧠 E2E тесты ⏱ 6h

`playwright` — ключевые user-flow:
- регистрация → вход → редактирование профиля → logout
- вход с неверным паролем
- восстановление пароля
- добавление в избранное
- фильтрация рецептов

Не критично пока, но сильно ускорит разработку когда появятся фичи.

### 6.7 🟢 🚀 Web Vitals мониторинг ⏱ 1h

`web-vitals` либа + отправка в `/dev/null` (пока без бэка), но в консоль — чтобы видеть LCP, CLS, INP и не допустить деградации.

## 🎯 Definition of Done

Для каждой задачи из backlog'а считается готовой когда:
- ✅ Реализован функционал
- ✅ Нет TS ошибок (`tsc --noEmit` проходит)
- ✅ Нет ESLint warnings на новый код
- ✅ Адаптив проверен на breakpoints: 320, 480, 768, 1024, 1440
- ✅ Клавиатурная навигация работает
- ✅ `prefers-reduced-motion` учтён
- ✅ Скриншоты до/после (для visual задач)
- ✅ Запись в CHANGELOG с объяснением «почему так»

---

## 🙏 P.S.

Стараемся не смешивать в одной итерации:
- 🚫 большой рефакторинг + новая фича (трудно ревьюить)
- 🚫 новые зависимости + существующий баг (неясно что сломало)
- 🚫 стили + логика в одном коммите (если не маленькие правки)

Правило: **одна итерация — одна смысловая единица работы.** Делаем маленькими понятными шагами — получаем консистентный продукт без неприятных сюрпризов в проде.
